# Admin Access Review Playbook

## Purpose

Review administrator access for network and security operations systems.

## Demo Identity

```text
Username: admin
Password source: browser-created demo account or approved environment variable
Demo IP: 192.168.1.1
```

## Review Steps

- Confirm the active administrator list.
- Check that shared accounts are limited to lab/demo scenarios.
- Validate that production systems use named accounts where possible.
- Confirm stale accounts are disabled or removed.
- Confirm privileged access is reviewed after role changes.

## Evidence To Capture

- Review date.
- Reviewer name.
- Systems reviewed.
- Accounts added, removed, or changed.
- Follow-up actions.
